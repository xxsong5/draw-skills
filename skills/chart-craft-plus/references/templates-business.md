# C 类：业务战略类模板

涵盖：商业模式画布、SWOT、竞品分析、价值链、用户旅程、产品路线图、甘特图、OKR 树、组织架构图、思维导图。

---

## 1 商业模式画布 (BMC)

### 布局结构

经典 9 宫格（5 列布局）：重要伙伴 | 关键活动+核心资源 | 价值主张 | 客户关系+渠道通路 | 客户细分。底部横贯：成本结构 | 收入来源。

### 必填字段（9 块）

重要伙伴、关键活动、核心资源、价值主张、客户关系、渠道通路、客户细分、成本结构、收入来源。

### 引导问题

- 产品/服务是什么？解决什么问题？
- 目标客户是谁？怎么触达？
- 靠什么赚钱？核心成本？
- 需要哪些合作伙伴？

### HTML 布局方式

Grid 5 列 + 底部 2 行合并列。每块有标题栏 + 要点列表。每块一个淡色背景区分。

## 2 SWOT 分析

### 布局结构

2x2 矩阵：S 优势（左上）/ W 劣势（右上）/ O 机会（左下）/ T 威胁（右下）。每块 3–5 条要点。

### 必填字段

每个象限至少 2 条。

### 引导问题

做得好 / 短板 / 外部机会 / 外部风险各是什么？

### HTML 布局方式

Grid 2x2。配色：S=emerald，W=slate/blue，O=amber，T=rose。

## 3 竞品分析图

### 布局结构

两种可选：

- **矩阵定位式**：2 轴坐标系（如易用性/功能丰富度），每个产品是一个气泡。
- **表格对比式**：行=维度，列=产品，星级/金额/描述评分。

### 必填字段

- 至少 2 个竞品
- 至少 3 个对比维度
- 每维评分/描述

### 引导问题

对比哪些产品？从哪些维度？各自表现？

### HTML 布局方式

矩阵：SVG 坐标轴 + 绝对定位气泡。表格：纯 HTML table + 星号/金额样式。

## 4 价值链图 (Value Chain)

### 布局结构

Porter 五大基础活动（入向物流 → 生产 → 出向物流 → 市场营销 → 服务）横向排列为粗箭头链，上方叠加支持活动（基础设施、人力资源、技术、采购）。

### 必填字段

- 每个基础活动的关键动作
- 支持活动的覆盖范围
- 价值增值说明

### 引导问题

- 业务的核心运营环节？
- 哪些环节创造最大价值？
- 哪些支持活动跨全链？

### HTML 布局方式

底部横向箭头链用梯形/平行四边形 CSS clip-path。上方支持活动用横条。尾部价值标记（margin/profit）。

## 5 用户旅程图

### 布局结构

横向时间阶段 + 纵向分层（阶段 / 用户行为 / 触点 / 情绪曲线 / 痛点 / 机会点）。

### 必填字段

- 至少 3 个阶段
- 每阶段用户行为
- 情绪曲线（高/低/中）
- 至少 1 个痛点

### 引导问题

用户经历几个阶段？每阶段在做什么？哪里体验好/差？改进机会？

### HTML 布局方式

Grid 表格行列。情绪曲线用 SVG path 平滑曲线。痛点红标，机会点绿标。

---

## 6 产品路线图 (Roadmap)

### 布局结构

横向时间轴（Q1–Q4 或 Now/Next/Later）+ 纵向功能线。每条功能线上用色块表示时间跨度，底部标里程碑菱形。

### 必填字段

- 时间跨度
- 至少 2 条功能线
- 每条线的时间区间和状态
- 至少 1 个里程碑

### 引导问题

时间跨度？功能线？每条线计划何时做？关键里程碑？

### HTML 布局方式

Grid 表格：行=功能线，列=时间单元。任务用圆角矩形横条。状态配色：已完成=实色，进行中=斜线，规划中=虚线边框。

---

## 7 甘特图 (Gantt)

### 布局结构

左侧任务列表 + 右侧时间横道。每条任务一根横条，带依赖箭头（可选）。可嵌套子任务分组。今天日期用红竖线。

### 必填字段

- 至少 5 个任务
- 每任务起止日期
- 任务层级（父子关系，若有）
- 任务负责人（可选）

### 引导问题

项目几个主阶段？每阶段任务？依赖关系？工期？

### HTML 布局方式

Grid：左列任务名 + 右侧按天/周列。横条用 absolute 定位 + calc 宽度。依赖用 SVG 画 L 形箭头。

## 8 OKR 对齐树

### 布局结构

顶部公司 Objective → 向下展开部门 Objectives → 再展开 KR（关键结果）。树状层级，箭头表示对齐关系。KR 带进度条。

### 必填字段

- 顶层 Objective
- 至少 2 个部门/团队 Objective
- 每个 O 下至少 2 个 KR
- KR 当前进度 %

### 引导问题

公司年度/季度 O 是什么？拆到哪些团队？每个团队的 KR 怎么定义？

### HTML 布局方式

树形 Flexbox。O 节点用粗框卡片，KR 用细框卡片 + 底部进度条。父子节点用 SVG 连线。

---

## 9 组织架构图 (Org Chart)

### 布局结构

树形，顶部 CEO/负责人，向下展开部门、二级部门、角色。每节点卡片（角色 + 人名 + 可选头像圈）。

### 必填字段

- 最顶层角色
- 至少 2 层级
- 每节点角色名

### 引导问题

顶层是谁？分几个部门？每部门下有哪些角色？

### HTML 布局方式

树形 Flexbox 横向排同级。SVG 画竖线+横线连父子。同级可用虚线框分组。

---

## 10 思维导图 (Mind Map)

### 布局结构

中心主题节点 → 向四周放射一级分支 → 每个分支再展开二级子节点。不同分支用不同颜色。

### 必填字段

- 中心主题
- 至少 3 个一级分支
- 每分支 2–3 个子要点

### 引导问题

核心主题？几个主方向？每方向要点？

### HTML 布局方式

中心节点绝对定位居中。一级分支按角度放射（CSS transform rotate）。连线用 SVG 曲线从中心出发。
