# B 类：流程逻辑类模板

涵盖：流程图、BPMN、泳道图、时序图、状态机图、决策树、数据流图。

---

## 1 流程图 (Flowchart)

### 布局结构

从上到下或从左到右的节点链。开始/结束节点（圆角矩形）、处理节点（矩形）、判断节点（菱形）、连线带箭头，分支带条件标签。

### 必填字段

- 流程标题
- 至少 3 个步骤节点
- 每个节点类型（开始/处理/判断/结束）
- 连线关系和分支条件

### 引导问题

- 起点和终点是什么？
- 中间关键步骤？
- 哪里有判断分支？条件是什么？

### HTML 布局方式

纵向 Flexbox 排节点。连线用 CSS 伪元素画竖线 + 三角箭头。分支处用 Grid 横排两条支路。

## 2 BPMN 图 (BPMN)

### 布局结构

横向流程图，使用标准 BPMN 符号：

- 开始事件（细圆环）
- 结束事件（粗圆环）
- 任务（圆角矩形）
- 网关（菱形，内嵌 X/+/○ 表示 XOR/AND/OR）
- 中间事件（双圆环）
- 泳池/泳道（顶部或左侧大标签容器）

### 必填字段

- 开始事件 + 结束事件
- 至少 3 个任务
- 至少 1 个网关
- 泳池/泳道划分

### 引导问题

- 涉及几个角色/系统？
- 流程的触发事件是什么？
- 有哪些决策点？

### HTML 布局方式

横向 Flexbox 主流程。SVG 绘制标准符号（圆环、菱形）。泳道用上下分栏 + 左侧角色标签。

---

## 3 泳道图 (Swimlane)

### 布局结构

横向或纵向分泳道，每道代表一个角色/部门。流程步骤（方块）落在对应泳道的时间位置上。跨道动作用带箭头连线。

### 必填字段

- 至少 2 个泳道（角色/部门）
- 至少 5 个步骤
- 步骤归属的泳道
- 跨道的交接点

### 引导问题

- 流程涉及哪些角色？
- 每一步谁做？
- 在哪一步做交接/传递？

### HTML 布局方式

CSS Grid：行=泳道，列=时间步骤。每个步骤卡片放在对应单元格。跨泳道线用 SVG 绘制在整体之上。

---

## 4 时序图 (Sequence Diagram)

### 布局结构

顶部：参与者横排（用户 / 前端 / 网关 / 服务 / DB）。每个参与者向下有虚线生命线。消息用带箭头横线连接两条生命线，标注方法名。激活期用竖条表示。

### 必填字段

- 至少 3 个参与者
- 至少 4 条消息
- 同步/异步区分（实线实心箭头 vs 虚线开箭头）
- 返回消息（虚线）

### 引导问题

- 有哪些参与方？
- 发起方先调谁？
- 每个调用是同步还是异步？
- 有没有失败/回滚分支？

### HTML 布局方式

SVG 为主：顶部参与者圆角矩形 + 向下竖虚线。消息箭头用 `<line>` + `<polygon>`。激活期用窄矩形覆盖生命线。**必须附图例**说明实线/虚线/激活含义。

---

## 5 状态机图 (State Machine)

### 布局结构

状态用圆角矩形节点，转换用带标签箭头连接。起始状态用黑点→箭头指入，终止状态用带圆环的黑点。可自环。

### 必填字段

- 至少 3 个状态
- 每个状态转换的触发条件/事件
- 起始状态和终止状态
- 自环（若有）

### 引导问题

- 对象有哪些生命周期状态？
- 什么事件触发状态转换？
- 有无状态回退？

### HTML 布局方式

节点用绝对定位或 Flexbox 自由排布。SVG 绘制箭头和曲线（自环用 arc path）。箭头标签放在连线中点。

---

## 6 决策树 (Decision Tree)

### 布局结构

根节点在顶部（问题），分支向下展开。每个分支用条件标签。叶子节点是结论/动作，用不同颜色区分。

### 必填字段

- 根问题
- 至少 2 层分支
- 每个分支的判断条件
- 叶子节点结论

### 引导问题

- 最核心的判断问题是什么？
- 每个判断下有几个可能结果？
- 每条路径最终会得到什么结论？

### HTML 布局方式

树形 Grid 布局。SVG 在父子节点间绘制带标签的斜线。叶子节点用不同背景色标注结论类型（通过=绿，拒绝=红，转人工=黄）。

---

## 7 数据流图 (Data Flow)

### 布局结构

从数据源（左）→ 处理节点 → 数据去向（右）。节点区分：Source（圆形）、Transform（方形）、Sink（梯形）、Store（圆柱）。箭头上标数据名/格式。

### 必填字段

- 至少 1 个数据源
- 至少 2 个处理步骤
- 至少 1 个数据去向
- 每条流的数据内容简述

### 引导问题

- 数据从哪来？
- 中间做了哪些加工？（清洗/聚合/转换）
- 最终去哪？（DB / 报表 / 下游服务）

### HTML 布局方式

横向 Flexbox 四栏（Source / Transform / Store / Sink）。节点用不同形状的 CSS（border-radius 区分圆/圆角/梯形）。箭头标签内联在 SVG path 上。**必须附图例**说明节点形状语义。
