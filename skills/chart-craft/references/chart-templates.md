# 图表模板参考

本文档定义每种图表的布局结构、必填字段和内容引导。生成图表时参考此文件。

---

## 1 流程图 (Flowchart)

### 布局结构

```
标题（居中，大字）
副标题（可选，描述流程目的）

┌─────────────────────────────┐
│         [开始节点]            │
│              │               │
│         [处理节点]            │
│           ╱    ╲             │
│     [判断]      [判断]       │
│       │           │          │
│    [处理]       [处理]       │
│       │           │          │
│         ╲    ╱              │
│         [结束节点]            │
└─────────────────────────────┘
```

### 必填字段

- 流程标题
- 至少 3 个步骤节点
- 每个节点的类型（开始/处理/判断/结束）
- 连线关系

### 内容引导问题

- 这个流程的起点和终点是什么？
- 中间有哪些关键步骤？
- 有哪些判断分支？每个分支的条件是什么？
- 有没有并行的步骤？

### HTML 布局方式

用 Flexbox 纵向排列节点，每个节点是一个 `<div>`，连线用 CSS 的 `::before` / `::after` 伪元素画竖线 + 箭头。分支用 Grid 布局横排两个分支。

## 2 架构图 (Architecture Diagram)

### 布局结构

```
标题

┌─── 前端层 ────────────────┐
│  [模块A]  [模块B]  [模块C] │
└────────────┬──────────────┘
             │
┌─── 服务层 ────────────────┐
│  [服务A]  [服务B]  [服务C] │
└────────────┬──────────────┘
             │
┌─── 数据层 ────────────────┐
│  [DB]    [缓存]   [队列]   │
└───────────────────────────┘
         │
┌─── 外部服务 ──────────────┐
│  [API-1]  [API-2]         │
└───────────────────────────┘
```

### 必填字段

- 系统名称
- 至少 2 个层级
- 每层的模块/组件
- 层级之间的连接关系

### 内容引导问题

- 系统分成几层？（前端/后端/数据/第三方）
- 每层有哪些模块？
- 模块之间怎么通信？（HTTP/消息队列/直接调用）
- 有没有外部服务依赖？

### HTML 布局方式

层级用纵向 Flexbox 排列，每层是一个带背景色的圆角矩形。模块用横向 Flexbox 排列在层内。层间用 SVG 画连接箭头。

## 3 ER 图 (ER Diagram)

### 布局结构

```
┌──────────┐         ┌──────────┐
│  用户     │───1:N──│  订单     │
├──────────┤         ├──────────┤
│ PK id    │         │ PK id    │
│    name  │         │ FK user_id│
│    email │         │    total │
│    phone │         │    status│
└──────────┘         └────┬─────┘
                          │1:N
                    ┌─────┴────┐
                    │  商品     │
                    ├──────────┤
                    │ PK id    │
                    │    name  │
                    │    price │
                    └──────────┘
```

### 必填字段

- 至少 2 个实体
- 每个实体的字段名
- 实体之间的关系类型（1:1 / 1:N / N:M）
- 主键标识

### 内容引导问题

- 系统里有哪些核心"东西"（实体）？
- 每个"东西"有什么属性？
- 它们之间什么关系？一对多还是多对多？

### HTML 布局方式

实体用绝对定位或 Grid 放置，每个实体是一个卡片（表头 + 字段列表）。关系线用 SVG `<line>` 绘制，两端标注基数。

## 4 商业模式画布 (Business Model Canvas)

### 布局结构

经典 5 列布局：

```
┌─────────┬─────────┬──────────────┬─────────┬─────────┐
│         │关键活动  │              │客户关系  │         │
│重要伙伴  │         │   价值主张    │         │客户细分  │
│         ├─────────┤              ├─────────┤         │
│         │核心资源  │              │渠道通路  │         │
├─────────┴─────────┴──────────────┴─────────┴─────────┤
│                    成本结构                            │
├──────────────────────────────────────────────────────-┤
│                    收入来源                            │
└───────────────────────────────────────────────────────┘
```

### 必填字段（九个区块）

1. **重要伙伴**：关键合作伙伴和供应商
2. **关键活动**：支持商业模式运行的主要活动
3. **核心资源**：实现价值主张所需的关键资源
4. **价值主张**：为客户解决的问题和独特价值
5. **客户关系**：与客户建立和维护的关系类型
6. **渠道通路**：与客户沟通和接触的渠道
7. **客户细分**：目标客户群体
8. **成本结构**：主要成本项
9. **收入来源**：收入流和支付方式

### 内容引导问题

- 产品/服务是什么？解决什么问题？
- 目标客户是谁？
- 怎么触达客户？
- 靠什么赚钱？
- 核心成本在哪？
- 需要哪些合作伙伴？

### HTML 布局方式

用 CSS Grid 实现 5 列布局。每列内部用 Flexbox 纵向排列区块。每个区块有标题栏 + 内容列表。成本和收入占满宽度，放在底部。

## 5 用户旅程图 (User Journey Map)

### 布局结构

```
┌──────────────────────────────────────────────────────┐
│ 阶段    │  认知     │  考虑     │  购买    │  使用    │
├─────────┼──────────┼──────────┼─────────┼──────────┤
│ 用户行为 │  ...      │  ...      │  ...    │  ...     │
├─────────┼──────────┼──────────┼─────────┼──────────┤
│ 触点     │  ...      │  ...      │  ...    │  ...     │
├─────────┼──────────┼──────────┼─────────┼──────────┤
│ 情绪曲线 │  ~~~^^~~~ │  ~vv~    │  ^^~~~  │  ~~~~^^  │
├─────────┼──────────┼──────────┼─────────┼──────────┤
│ 痛点     │  ...      │  ...      │         │  ...     │
├─────────┼──────────┼──────────┼─────────┼──────────┤
│ 机会点   │  ...      │  ...      │  ...    │  ...     │
└─────────┴──────────┴──────────┴─────────┴──────────┘
```

### 必填字段

- 至少 3 个阶段
- 每个阶段的用户行为
- 情绪曲线（高/低/中性）
- 至少标注 1 个痛点

### 内容引导问题

- 用户从接触到完成目标，经历了哪些阶段？
- 每个阶段用户在做什么？接触了什么？
- 哪些阶段体验好？哪些差？
- 有什么改进机会？

### HTML 布局方式

用 CSS Grid 实现行列表格。情绪曲线行用 SVG `<path>` 画平滑曲线。痛点用红色标记，机会点用绿色标记。

## 6 思维导图 (Mind Map)

### 布局结构

```
                    ┌─ 子主题1
          ┌─ 主题A ─┤
          │         └─ 子主题2
中心主题 ──┤
          │         ┌─ 子主题3
          └─ 主题B ─┤
                    └─ 子主题4
```

### 必填字段

- 中心主题
- 至少 3 个一级分支
- 每个分支 2-3 个子要点

### 内容引导问题

- 核心主题是什么？
- 有几个主要方向？
- 每个方向下面有哪些要点？

### HTML 布局方式

中心节点绝对定位在页面中央。一级分支用 CSS transform 按角度放射分布。连线用 SVG `<line>` 从中心连到各分支。

## 7 竞品分析图 (Competitive Analysis)

### 布局结构

两种布局可选：

**矩阵定位式：**
```
              高
    功能丰富度 │  [产品A]
              │       [产品C]
              │  [产品B]
    低 ───────┼──────── 高
              │        易用性
              │
              低
```

**表格对比式：**
```
┌──────────┬────────┬────────┬────────┐
│  维度     │ 产品A  │ 产品B  │ 产品C  │
├──────────┼────────┼────────┼────────┤
│  功能     │  ★★★   │  ★★    │  ★★★   │
│  价格     │  $$    │  $     │  $$$   │
│  易用性   │  ★★    │  ★★★   │  ★     │
└──────────┴────────┴────────┴────────┘
```

### 必填字段

- 至少 2 个竞品
- 至少 3 个对比维度
- 每个维度的评分/描述

### 内容引导问题

- 要对比哪些产品？
- 从哪些维度对比？（功能/价格/体验/增长）
- 每个维度上各自表现如何？

## 8 SWOT 分析 (SWOT Analysis)

### 布局结构

```
┌──────────────┬──────────────┐
│              │              │
│   S 优势     │   W 劣势     │
│   (内部正面)  │   (内部负面)  │
│              │              │
├──────────────┼──────────────┤
│              │              │
│   O 机会     │   T 威胁     │
│   (外部正面)  │   (外部负面)  │
│              │              │
└──────────────┴──────────────┘
```

### 必填字段

- S：至少 2 条优势
- W：至少 2 条劣势
- O：至少 2 条机会
- T：至少 2 条威胁

### 内容引导问题

- 做得好的地方有哪些？（优势）
- 短板在哪里？（劣势）
- 外部有什么有利因素？（机会）
- 外部有什么风险？（威胁）

### HTML 布局方式

CSS Grid 2x2 布局。优势区块绿色调，劣势蓝/灰调，机会黄/橙调，威胁红/粉调。每个区块标题突出，要点用列表排列。

## 9 产品路线图 (Product Roadmap)

### 布局结构

```
┌─────────────────────────────────────────────────────┐
│         Q1          Q2          Q3          Q4       │
├──────────┬──────────┬──────────┬────────────────────┤
│ 模块A    │ ██████   │ ████████ │          │         │
├──────────┼──────────┼──────────┼────────────────────┤
│ 模块B    │          │ ████     │ ████████ │         │
├──────────┼──────────┼──────────┼────────────────────┤
│ 模块C    │          │          │ ████     │ ████████ │
├──────────┴──────────┴──────────┴────────────────────┤
│  ◆ v1.0       ◆ v1.5       ◆ v2.0      ◆ v2.5     │
└─────────────────────────────────────────────────────┘
```

### 必填字段

- 时间轴（季度或月份）
- 至少 2 个模块/功能线
- 每个模块的时间区间
- 至少 1 个里程碑

### 内容引导问题

- 时间跨度多大？（季度/半年/一年）
- 有几条功能线？
- 每条功能线在什么时间段做？
- 有哪些关键里程碑？

### HTML 布局方式

横向时间轴用 Grid。功能线纵向排列，每条线内的任务用圆角矩形横条表示时间跨度。不同状态用不同颜色/图案（实心=已完成，斜线=进行中，虚线=规划中）。里程碑用菱形标记。

## 10 组织架构图 (Org Chart)

### 布局结构

```
              ┌─────────┐
              │  CEO     │
              └────┬─────┘
           ┌───────┼───────┐
      ┌────┴───┐  │  ┌────┴───┐
      │ CTO    │  │  │ CMO    │
      └────┬───┘  │  └────────┘
     ┌─────┼─────┐│
 ┌───┴──┐ │ ┌────┴┤
 │前端TL│ │ │后端TL│
 └──────┘ │ └─────┘
```

### 必填字段

- 最顶层角色
- 至少 2 层级
- 每个角色名称
- 汇报关系连线

### 内容引导问题

- 团队的顶层角色是谁？
- 下面分几个方向/部门？
- 每个方向下有哪些角色？
- 角色之间的汇报关系？

### HTML 布局方式

树形结构，每层用 Flexbox 横向排列。父子关系用 SVG 竖线 + 横线连接。同级用虚线框分组。每个节点是一个小卡片（角色 + 可选人名）。
